# 本地 AI 音乐 MV 工程系统

这是可复用的方法。

目标是把任何一首歌变成一条可重复的生产线：

```text
歌曲 -> 情绪地图 -> 视觉世界 -> 素材库 -> 镜头计划 -> 生成短片 -> 剪辑 -> 复盘 -> 修正后成片
```

## 五层结构

### 1. 歌曲分析

先搞清楚歌在干什么，才有可能做出有力量的 MV。

要提取：

- BPM 或大致速度
- 前奏 / 主歌 / 预副歌 / 副歌 / 桥段 / 结尾
- 情绪曲线
- 最强歌词意象
- 节拍落点和转场点

### 2. 视觉世界

给这首歌选一个统一的世界。

定义：

- 主角
- 服装语言
- 环境语言
- 颜色方案
- 光线逻辑
- 镜头个性

### 3. 素材库

先做参考图，再做视频。

最低配置：

- 角色身份图
- 服装图
- 舞台 / 环境图
- 局部特写图
- 标题 / Logo 图
- 音乐参考

### 4. 镜头工厂

一镜一做，不要把整支 MV 扔给一次生成。

每个镜头都要有：

- 参考图
- 动作
- 镜头运动
- 光线
- 时长
- 淘汰标准

### 5. 复盘循环

每一个失败输出都变成一条规则。

记录：

- 哪儿失败了
- 为什么失败
- 哪个修复 prompt 有效
- 这个镜头该重做还是直接砍掉

## 为什么能迁移到第二首歌

你保留的是工程系统，只替换：

- 歌曲分析
- 视觉世界
- 素材参考
- 镜头表

生产纪律保持不变。
